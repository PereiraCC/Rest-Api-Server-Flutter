This is the REST API of my company management. This was developed in node js with express framework, it connects with firebase as a database. You can find the source code here you can also contact for an upgrade request.


USERS

This is the users path where it contains the following actions:

  1. GET ALL USERS: Get all registered users with pagination

    2. GET: https://rest-server-agents-pereira.herokuapp.com/api/users?limit=10&from=1

    • Params
      1. limit (Number)
      2. from (Number)
    • No require authorization (No JWT)

  2. GET A USER: Get a specific user by means of their id

    3. GET: https://rest-server-agents-pereira.herokuapp.com/api/users/001

    • Params
      1. Identification of user
    • Require authorization

  3. CREATE NEW USER

    4. POST: https://rest-server-agents-pereira.herokuapp.com/api/users

    • No require params
    • Require JSON data with new data
    • No require authorization

  4. UPDATE A USER

    5. PUT: https://rest-server-agents-pereira.herokuapp.com/api/users/001

    • Params
      1. Identification of user
    • Require JSON data
    • Require authorization

  5. DELETE A USER

    6. DELETE: https://rest-server-agents-pereira.herokuapp.com/api/users/001

    • Params
      1. Identification of user
    • Require authorization

AUTHENTICATION

This is the authentication path where it contains the following actions:

  1. AUTHENTICATION MANUAL: Log in with email and password

    2. POST: https://rest-server-agents-pereira.herokuapp.com/api/auth/login

    • No require params
    • Require JSON data (Email and password)
    • No require authorization (No JWT)

  2. AUTHENTICATION WITH GOOGLE: Log in with google sing in

    3. POST: https://rest-server-agents-pereira.herokuapp.com/api/auth/google

    • No require params
    • Require JSON data (google id_token)
    • No require authorization

  3. JSON WEB TOKEN (JWT) VALIDATION

    4. POST: https://rest-server-agents-pereira.herokuapp.com/api/auth/validJWT

    • No require params
    • Require JSON data (token)
    • No require authorization

UPLOADS IMAGES

This is the uploads path where it contains the following actions:

Allowed extensions

  • png
  • jpg
  • jpeg
  • gif
  1. UPLOAD IMAGE FOR A USER

    2. PUT: https://rest-server-agents-pereira.herokuapp.com/api/uploads/users/001

    • Params:
      1. Identification of a user
    • Require file (Image)
    • Require authorization

  2. UPLOAD IMAGE FOR AN AGENT

    3. PUT: https://rest-server-agents-pereira.herokuapp.com/api/uploads/agents/017?userID=123ABC

    • Params:
      1. Identification of an agent
      2. Identification of a user (userID)
    • Require file (Image)
    • Require authorization

  3. UPLOAD IMAGE FOR A PRODUCT

    4. PUT: https://rest-server-agents-pereira.herokuapp.com/api/uploads/products/408?userID=123ABC

    • Params:
      1. Identification of a product
      2. Identification of a user (userID)
    • Require file (Image)
    • Require authorization
