Users

This is the users' path where it contains the following actions:

  1. GET api/users: Get all registered users
    • No require params
    • No require authorization

  2. GET:ID api/users: Get a specific user by means of their id
    • Params: Identification of a user
    • Require authorization

  3. POST api/users: Create new user
    • No require params
    • Require JSON data
    • No require authorization

  4. PUT: api/users: Update an user
    • Params: Identification of a user
    • Require JSON data
    • Require authorization

  5. DELETE: api/users: Delete an user
    • Params: Identification of a user
    • Require authorization

Authentication

This is the authentication path where it contains the following actions:

  1. POST api/auth/login: Log in with email and password
    • No require params
    • Require JSON data
    • No require authorization

  2. POST api/auth/google: Log in with google sing in
    • No require params
    • Require JSON data (google id_token)
    • No require authorization

Uploads images

This is the uploads path where it contains the following actions:

  1. PUT:ID api/uploads/users: Upload image for a user
    • Params: Identification of a user
    • Require file (Image)
    • Require authorization

  2. PUT:ID api/uploads/agents: Upload image for an agent
    • Params: Identification of a agent
    • Require file (Image)
    • Require authorization
